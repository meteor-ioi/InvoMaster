# 视觉设计规范 (Style Guide)

本文档定义了 HITL Document Extraction 项目的视觉语言与设计原则，确保全站 UI 在白天与黑夜模式下的高度一致性与高级感。

## 1. 核心设计理念

项目采用 **Modern Glassmorphism (现代玻璃拟态)** 风格，结合柔和的梯度背景与半透明层级，营造出通透、轻盈且具有层次感的界面。

- **层级感**：通过背景梯度、背景虚化（Backdrop Blur）和阴影来区分容器。
- **动态呼吸**：所有交互（Hover, Active）应伴随平滑的过渡效果（0.3s cubic-bezier）。
- **主题感知**：组件应根据 `theme` 变量动态调整，而非硬编码颜色。

---

## 2. 配色方案

项目主要通过 CSS 变量进行颜色管理，严禁在组件内直接使用硬编码的 RGB(A) 或 HEX 值（特殊微调除外）。

### 2.1 基础色调 (Base)

| 变量名 | 类别 | 白天模式 (Light) | 黑夜模式 (Dark) | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| `--bg-gradient` | 页面背景 | #f1f5f9 → #e2e8f0 | #1e293b → #0f172a | 径向梯度，固定背景 |
| `--glass-bg` | 卡片主体 | `rgba(255, 255, 255, 0.8)` | `rgba(30, 41, 59, 0.7)` | 配合 12px 模糊 |
| `--glass-border` | 边框线 | `rgba(0, 0, 0, 0.1)` | `rgba(255, 255, 255, 0.1)` | 极细边框 |
| `--input-bg` | 输入/预览背景 | `rgba(0, 0, 0, 0.03)` | `rgba(255, 255, 255, 0.05)` | 用于预览块和输入框 |

### 2.2 文字颜色 (Typography)

- **Primary**:
  - 白天: `#0f172a` (深蓝黑)
  - 黑夜: `#f8fafc` (近纯白)
- **Secondary**:
  - 白天: `#64748b` (深灰蓝)
  - 黑夜: `#94a3b8` (冷灰色)

### 2.3 功能色 (Functional)

- **Primary (品牌主色)**: `#3b82f6` (活力蓝)
- **Accent (强调色)**: `#8b5cf6` (紫罗兰)
- **Success (成功色)**: `#10b981` (翠绿色)

## 3. 组件设计原则

### 3.1 玻璃卡片 (Glass Card)
- **圆角**: 统一 `16px`。
- **背景**: `var(--glass-bg)`。
- **模糊**: `backdrop-filter: blur(12px)`。
- **阴影**: 
  - 白天: `0 8px 24px 0 rgba(148, 163, 184, 0.2)`
  - 黑夜: `0 8px 32px 0 rgba(0, 0, 0, 0.37)`

### 3.2 内容预览区域 (Content Preview)
用于展示 JSON, XML, Markdown 或表格。
- **背景原则**：
  - 白天模式下，背景应尽可能轻薄（或与容器融合），通常使用 `var(--input-bg)`。
  - 黑夜模式下，使用稍微偏暗的层级 (`rgba(0,0,0,0.2)`)。
- **文字**：对于代码类预览，白天模式使用 `var(--text-primary)`，黑夜模式可使用 `#cbd5e1` 提高对比度。

### 3.3 数据表格 (Data Tables)
- **斑马纹 (Zebra Stripes)**：
  - 白天: `rgba(0, 0, 0, 0.01)`
  - 黑夜: `rgba(255, 255, 255, 0.02)`
- **边框**: 内部单元格应由 `1px solid var(--glass-border)` 分隔。
- **Hover**: 悬停时背景应有微弱变亮或变暗反馈。

### 3.4 按钮 (Buttons)
- **渐变按钮**: 使用 `linear-gradient(135deg, var(--primary-color), var(--accent-color))`。
- **图标按钮**: 使用 `var(--input-bg)` 背景，Hover 时切换至 `var(--glass-border)`。

---

## 4. 后续开发指导

1. **优先使用变量**：在编写 style 时，首选 `var(--...)` 指向定义的 CSS 变量。
2. **处理主题切换**：组件必须接收 `theme` 属性或监听 `[data-theme]` 属性的变化，并据此决定内部非 CSS 变量覆盖的逻辑（如代码高亮颜色）。
3. **保持圆角一致**：大容器 16px，小组件（按钮、标签）8px-10px。
4. **添加微动画**：所有的出现、消失和状态切换，都应当配合 `slideUp`, `fadeIn` 等预设动画。
